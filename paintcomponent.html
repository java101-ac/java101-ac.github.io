Note: For the following exercises, assume g stores a Graphics object.

1. What does g.drawOval(0,0,50,50); display?

    A. A circle of radius 25 pixels in the top left-hand corner
    B. An oval of radius 50 pixels
    C. A circle of radius 0 positioned 50 pixels from the top and left
    D. An oval of radius 25 pixels

2. What does the code below display in the JFrame?

          int[] x = {10,30,60,100,150};
  	      int[] y = {20,40,70,110,160};
  
  	      g.fillPolygon(x,x,3);
    A. A line segment
    B. A filled polygon consisting of 5 vertices
    C. A filled polygon consisting of 3 vertices
    D. Nothing, an error will prevent execution

3. Will the code below display identical arcs on the JFrame? Why or why not?

        g.drawArc(50,50,20,20,-180,270);
  	    g.drawArc(50,50,20,20,90,-270);
    A. Yes. The angle drawn by both lines of code are geometrically equivalent to each other
    B. Yes. The arcAngle of the second line of code is identical, just in a different direction
    C. No. The startAngle and arcAngle parameters are different, so a different arc will be drawn
    D. No. The arcs drawn start and end at opposite points, so they will not be identical

4. What is the purpose of using a try-catch block with Scanner and PrintWriter?

    A. The try-catch block can alert you of a file-not-found error
    B. Scanner and PrintWriter will not function properly without a try-catch block
    C. A try-catch prevents large files from being corrupted during parsing
    D. A try-catch block optimizes the performance of Scanner and PrintWriter
